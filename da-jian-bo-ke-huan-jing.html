搭建博客环境

Date 2015-12-10(星期四) 10:32 Tags Python / Pelican / Publishing

搭建环境

    #下载python3、pip3
    #创建blog的虚拟环境
    mkdir git_blog
    cd git_blog
    virtualenv --no-site-packages venv
    #进入虚拟环境
    source venv/bin/activate
    python3 install pelican
    python3 install markdown
    #创建pelican的框架，根据提示操作，后面也可以在生成的 pelicanconf.py 文件中修改
    pelican-quickstart
    #创建的项目结构注释
    git_blog/
    |-- content                         #写好的Markdown格式的博文
    |     └-- (pages)                  #Markdown格式的静态界面(比如About，同WP的Page)
    |-- output                           #通过Pelican最终生成的html界面
    |-- develop_server.sh          #Pelican服务器的脚本
    |-- fabfile.py                       #生成和上传等自动化操作
    |-- Makefile                        #make的自动化脚本
    |-- pelicanconf.py                #主配置文件(测试环境)
    |-- publishconf.py               #主配置文件的扩展，用于(生产环境)

pelicanconf.py 中的常用配置

    AUTHOR = u'MuYiFC' #博客作者
    SITENAME = u'' #博客标题
    SITESUBTITLE = u'我本微末凡尘，可也心向天空' #博客子标题
    SITEURL = '' #主页地址
    DEFAULT_DATE='fs'#快速获取本地时间作为文章发表时间
    DEFAULT_DATE_FORMAT = '%Y %B %d %a' #使用“年月日时”的时间显示方式
    TIMEZONE = 'Asia/Shanghai' #默认时区，亚洲上海，其他时区可查看国际标准
    DEFAULT_LANG = u'zh' #默认语言为中文
    USE_FOLDER_AS_CATEGORY = True #将**content**下的文件夹作为分类，博主目前测试仅可为md文件的所在目录，不支持子分类
    PLUGIN_PATH = "D:\Poetic_Being\MyBlog\MyPlugins" #插件路径声明
    PLUGINS = ["assets"] #插件使用列表
    THEME = 'gum' #使用主题

第一篇博文

使用markdown写博文，其中需要一些固定的格式，作为博客的索引

    Title: My super title  #文章标题
    Date: 2015-12-10 10:50 #文章时间，如果前面设定过可忽略
    Tags: thats, awesome #文章标签
    Category: yeah #文章分类
    Author: MuYiFC #文章作者，缺省为博客作者
    Summary: Short version for index and feeds #文章概括
    Slug:post  #如果博文标题冲突时，可以设置该值不同来解决，例如需要相同标题不同版本的博文，默认不用设置

    其他内容就是正文了，按markdown格式写就可以了

本机预览

    make publish
    make serve

之后就可以在 127.0.0.1:8000 预览 需要停止服务器，可以直接用 CTRL+c 停止进程或是调用

    ./develop_server.sh stop

部分常用的pelican的操作指令

    make html              #创建静态界面，将 content 中的文件编译生成到 output 目录中
    make regenerate     #自动生成静态界面，如果频繁更改，使用该命令可以在修改html后自动生成html，相当于执行 make html
    make serve             #创建服务器
    make devserver       #相当于 regenerate+serve
    make publish           #生成用于发布的html，准备正式发布时使用

代码上传

提交代码到git，官方教程

    cd git_blog/output/
    git init
    #创建没有父节点的分支节点，git默认需要有个 gh-pages 的节点才能生成网页，master 节点不会生效
    git checkout --orphan gh-pages
    #拉取远程代码
    git pull -u origin gh-pages
    #提交
    git push -u origin gh-pages

后续push该目录内的内容即可 访问项目地址可以在该git项目Settings/GitHub Pages中找到

变更github的域名

    # 添加两个A记录
    @ A 192.30.252.153
    @ A 192.30.252.154
    # 一个CNAME
    blog CNAME muyifc.github.io.
    # 在 output 文件夹中添加 CNAME 文件，添加域名 blog.xmdreamsky.com
    # 上传到git
    # git的库名不需要 <git账号>.github.io，直接随意设定一个就可以，如 blog

变更主题

    #该地址会下载一大部分主题
    git clone https://github.com/getpelican/pelican-themes.git
    #加载主题，该命令将指定的主题复制到虚拟库中 /venv/lib/python3.5/site-packages/pelican/themes/<themename>
    pelican-themes -i pelican-themes/<themename>
    # 修改pelicanconf.py，添加 THEME = '<themename>'，重新生成即可生效
    #如果要修改主题，需修改 pelican-themes/<themename>/templates 中的对应模板文件，再刷新模板
    pelican-themes -U pelican-themes/<themename>
    #删除主题
    pelican-themes -r <themename>

添加评论

在Disqus上申请一个站点，记牢Shortname。这个 Shortname 就是你注册的用户名，在pelicanconf.py添加

    DISQUS_SITENAME = Shortname
